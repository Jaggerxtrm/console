## Problems Encountered

| Problem | Root Cause | Resolution |
|---------|-----------|------------|
| Multi-blocker issues rendered more than once. | The blocker tree inserted a blocked issue under every unresolved blocker, making the feed a DAG rendered as a duplicate tree. | Added primary-blocker selection: same parent/epic first, then priority, recent update, stable id. Other blockers remain visible as relationship chips. |
| `blocks` / `blocked_by` semantics were inconsistent in the UI. | Beads data exposed dependency orientation differently than the initial UI assumption, and `blocks` was temporarily removed from status derivation. | Restored blocked status derivation for both unresolved blocker dependency forms while normalizing display placement. |
| Epic marker coloring was lost through nested blocker chains. | Relation state only distinguished direct child vs blocked child, so children inside an epic subtree could fall back to generic/blocker coloring. | Propagated `epic` relation through descendants so epic subtree markers stay purple even when an internal child blocks another issue. |
| Right-column metadata became visually noisy. | Type icons, independent priority colors, uppercase status, and slash separators made the row uneven and too colorful. | Reworked row metadata to `P<n> TYPE status date`, with only priority/type colored by type and all text at the same compact size. |
| Relationship chips did not show useful title tooltips. | The parent second-line element had its own `title`, and chip fallback used only dependency title rather than resolving `issueById`. | Removed parent `title` and resolved chip tooltips from `issueById` first, then dependency title, including `P<n> TYPE` when available. |
| `relates-to` created structural nesting. | `getParentId` treated `relates-to` like `parent-child`, incorrectly making non-blocking references into tree parents. | Restricted structural nesting to `parent_id`, `parent-child`, and legacy `parent`; `relates-to` is display-only. |

| Memory gate initially blocked close. | Combined `bd kv set`/`bd close` command obscured the ack state and quoting behavior. | Set `memory-acked:<id>` explicitly, verified with `bd kv get`, then closed the issue. |

## Code Changes

### Unified shell and sidebar

- `apps/gitboard/src/dashboard/components/shell/{Sidebar,TopBar,MainPane}.tsx`, `apps/gitboard/src/dashboard/stores/shell.ts`, and `apps/gitboard/src/types/shell.ts` implement the repo-centric shell: GitHub/Beads surfaces, tab strip, repo selection, collapse state, and default Beads tab now set to Feed.
- `apps/gitboard/src/dashboard/hooks/useRepoTree.ts`, `apps/gitboard/src/api/routes/beads.ts`, `apps/gitboard/src/core/github-store.ts`, and `apps/gitboard/src/types/{beads,github}.ts` support real activity sorting across GitHub/Beads repos, including Beads `last_activity_at`.
- `apps/gitboard/src/dashboard/styles/globals.css` received the black-background/dense-row shell styling used across GitHub timelines and Beads rows.

### Beads Feed

- `apps/gitboard/src/dashboard/components/beads/IssueFeed.tsx` is now the central agent workflow view. It renders compact two-line rows: title line plus a second line for priority/type/status/date, child count, grouped relationship chips, PR, and agent metadata.
- Feed sections are `▸/▾ in_progress`, collapsible `open` expanded by default, and collapsed `closed` by default. In-progress issues duplicate into the Open graph until closed, preserving active graph context.
- The dependency graph display supports `blocks`, `blocked_by`, `tracks`, `related`, `relates-to`, `parent-child`, `discovered-from`, `until`, `caused-by`, `validates`, and `supersedes`. Unknown future types render after known types in deterministic order.
- Tree rendering avoids duplicate multi-blocker rows via primary-blocker placement while preserving all relationships in the second-line chip groups.
- `relates-to` is display-only; only `parent_id`, `parent-child`, and legacy `parent` create structural nesting.
- Epic child markers propagate through nested descendants, including internal blocker edges, so epic context remains visually stable.

### Beads tabs

- `apps/gitboard/src/types/shell.ts` and `apps/gitboard/src/dashboard/components/beads/BeadsRepoView.tsx` removed Kanban and Closed tabs from the Beads tab strip. Feed/Triage/Memories remain, with Closed now represented as a collapsed Feed section.
