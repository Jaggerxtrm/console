## Code Changes

### Change summary

15 commits ahead of pre-session `origin/main` (`dc01ac4`), merged to main as `1fff60a` via PR #9. Net effect: typography rescale, viewport fix, card enclosure system, per-entry expanded view styling, beadboard column-width fix.

### Key modifications

**Token system** (`packages/ui/src/styles/tokens.css`):
- Type scale: `--text-xs/sm/base/md/xl` now in rem (`0.75/0.8125/0.875/1/1.25rem`) — maps to 12/13/14/16/20px at 16px root.
- New card tokens: `--card-border`, `--card-radius-sm: 6px`, `--card-radius-md: 8px`, `--card-padding: 12px`. Existing `--radius-*` defaults left at `0` to preserve flat aesthetic elsewhere.

**Gitboard shell** (`apps/gitboard/src/dashboard/styles/globals.css`, `App.tsx`, `RepoSidebar.tsx`):
- `body { overflow-x: hidden; overflow-y: auto }`, added `html { font-size: 16px }` baseline.
- Sidebar widths now `clamp(160px, ${width}px, 480px)`; `min-width: 0` on flex children to prevent overflow at zoom.
- No more horizontal scrollbar at 533–1600px viewports.

**Card enclosure** (`globals.css`, `ActivityTimeline.tsx`, `EventDetail.tsx`):
- Row-level enclosure on `.timeline-row`, `.pr-row`, `.issue-row` (3-sided border, 6px radius, tinted bg).
- Per-entry card pattern on `.pr-conversation-entry` (commit/comment/review/body): `--surface-secondary` bg, 10/14 base padding, 14/16 for body/comment/review variants, `--card-radius-md`, 8px inter-entry gap.
- `.pr-expanded-body` / `.issue-expanded-body` themselves stay flat (no outer frame).

**Beadboard fix** (`apps/beadboard/src/dashboard/components/beads/StatusColumn.tsx`):
- Replaced `minWidth:240, maxWidth:300` with `flex: "1 1 0", minWidth: 260`. Columns now share parent width equally.
