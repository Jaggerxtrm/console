## Code Changes

### Gitboard UI and Design System

- `apps/gitboard/src/dashboard/App.tsx` now detects `/gitboard/design-preview`, renders a separate design preview surface, applies the `westworld-app` shell class to the real app, and changes the top-left brand to `xtrm.wtf`.
- `apps/gitboard/src/dashboard/styles/globals.css` now contains the grounded cinematic FUI visual system: dark graphite/navy palette, low-opacity grid, cyan edge lighting, denser system partition sidebar, consistent topbar/tab/header sizing, operational row states, and PR conversation styling. Several iterative passes tightened shape consistency and reduced decorative SaaS/cyberpunk cues.
- `ActivityTimeline`, `GithubPanel`, `RepoSidebar`, and `StatsHeader` received class hooks so CSS can target real components consistently instead of relying only on brittle inline-style selectors.

### GitHub Data and Pull Request Functionality

- `apps/gitboard/src/api/routes/github.ts` adds `githubApi`/paginated GitHub REST helpers and the new PR detail endpoint: `/api/github/prs/:owner/:repo/:number/detail`.
- `apps/gitboard/src/types/github.ts` adds PR detail types for issue comments, reviews, review comments, commits, changed files, timeline events, and partial endpoint errors.
- `apps/gitboard/src/dashboard/lib/client.ts` adds `getPrDetail`.
- `apps/gitboard/src/dashboard/hooks/useGithubActivity.ts` raises PR fetch limit to 1000 to avoid repo-heavy first-page bias.
- `apps/gitboard/src/dashboard/components/github/GithubPanel.tsx` filters PRs/issues by active selected repo as well as owner.
- `apps/gitboard/src/dashboard/components/github/PrTimeline.tsx` was rewritten into a richer inline/collapsible PR view: state pills, compact PR summary, conversation-style thread, GitHub detail hydration, comments/reviews/review-comments/commits/files/timeline rendering, and no internal scroll box for PR description.

### Runtime / xtrm Initialization

- `.claude/skills` was moved aside and recreated as a symlink to `../.xtrm/skills/active` by `xt init --yes`.
- `.xtrm/`, `.pi/`, `AGENTS.md`, `CLAUDE.md`, `.mcp.json`, and generated hook/settings files were created or updated by `xt init`.
- `.env` was updated locally with `GITHUB_TOKEN` for Podman/compose runtime; `.env` is gitignored and must not be committed.
