## Code Changes

### Realtime contract module (forge-dgd → commit 8d24ba7)

- `apps/gitboard/src/types/realtime.ts` (NEW) — `REALTIME_PROTOCOL_VERSION=1`, `RealtimeEnvelope<E,D>` typed envelope, `GithubRealtimeEvent` + `BeadsRealtimeEvent` union types, per-event payload aliases. **Architectural note**: `version: string` was a contract drift discovered in forge-f0g review (originally `number` with `Date.parse()` fallback that silently coerced Dolt commit hashes to a default). Forge-f0g amended the type before merge.
- `apps/gitboard/src/api/ws/channels.ts` — `ChannelRegistry.publish` stamps `seq` + `ts` + `version` + `boot_id` automatically; per-channel ring buffer (constant `RING_SIZE=500`); `replay(channel, since_seq, last_boot_id)` helper.
- `apps/gitboard/src/api/ws/handler.ts` — handles `resume` action: replays buffered envelopes when boot_id matches AND seq is in range; emits domain `:sync_hint` envelope on mismatch. Subscribe with mismatched protocol major closes with code 4001.
- `apps/gitboard/src/dashboard/lib/ws.ts` — client tracks last seq + boot_id per channel; auto-sends resume on reconnect with the previously seen markers.

### Live GitHub updates (forge-f0g → commit 4cd9666)

- `apps/gitboard/src/core/github-poller.ts` — `pollIssues` + `pollPullRequests` with `since=watermark` + ETag (in-memory) + adaptive cadence (1m active / 10m / 30m idle) + rate-limit backoff (<500 remaining → 60s pause). **Known regression introduced here**: page-loop dropped (was page=1,2,... while items.length === 100); now single `per_page=100` request only. Caught by Codex post-merge → forge-yy27 Fix 2 + Fix 3.
- `apps/gitboard/src/core/github-store.ts` + `apps/gitboard/src/core/store.ts` — `repo_poll_state` table for per-(repo, endpoint) ETag + watermark persistence across server restarts.
- `apps/gitboard/src/dashboard/stores/github.ts` — `upsertPr`/`upsertIssue` actions (replace by `(repo, number)` or prepend; preserve newest-first sort).
- `apps/gitboard/src/dashboard/hooks/useGithubActivity.ts` — switched from legacy `new_event`/`new_commits` to `github:event.append`/`pr.upsert`/`issue.upsert`/`sync_hint`. **Known regression**: `sync_hint` reload path gated on `'repos' in msg.data` but server emits `{reason, channel, since_seq}` without repos. Caught by Codex → forge-yy27 Fix 4.

### Live Beads updates (forge-igg → commit b854872)

- `apps/beadboard/src/core/beads-change-watcher.ts` (NEW) — per-project watcher; active 2s / idle 10s poll; coalesce 1500ms / 50 items / 16KB; emits `beads:issue.*`/`dep.*`/`memory.*`/`kv.*`/`batch`/`sync_hint`/`source_health` envelopes via ChannelRegistry. Diffs four entity snapshots per Dolt commit-hash tick. Semantic alias detection (deferred/superseded/flagged/unflagged) derived from field transitions vs previous snapshot.
- `apps/beadboard/src/core/dolt-client.ts` — `+ getCommitHash()` / `getIssuesSince()` / `fetchIssuesSince()` (additive; existing public API signature-compatible).
- `apps/beadboard/src/core/project-scanner.ts` — `+ probeSourceHealth` helper.
- `apps/gitboard/src/api/server.ts` — starts `BeadsChangeWatcher` with the active `ChannelRegistry` on server boot.
- `apps/beadboard/src/dashboard/stores/beads.ts` — `+ deps`/`kvs` state + `upsertDep`/`removeDep`/`upsertMemory`/`removeMemory`/`upsertKv`/`removeKv`/`applyBatch`/`applySyncHint`/`setSourceHealth` actions.
- `apps/beadboard/src/dashboard/hooks/useBeadsLive.ts` (NEW) — subscribes `beads:changes`; routes every event to the matching store action. **Known regression**: filters on `msg.projectId` at envelope top level instead of `msg.data.projectId` → live Beads sync silently dead in production. Caught by Codex → forge-yy27 Fix 1.

### Dolt connection stability (forge-1qz → commit 20f60eb)

- `apps/beadboard/src/core/dolt-client.ts` — migrated from `mysql.createConnection` to `mysql.createPool({connectionLimit: POOL_SIZE=4, idleTimeout: 60s, enableKeepAlive: true})`. Module-singleton `runtime` state. `SELECT 1` probe every `PROBE_INTERVAL_MS=5000ms`. Per-query `withTimeout(promise, QUERY_TIMEOUT_MS=3000)` wrapper records metric. Circuit breaker opens after `BREAKER_THRESHOLD=5` consecutive failures; `breakerOpenUntil = now + backoffDelay(failures)` (250ms→30s exponential + ±20% jitter). Recovery via probe-then-retry — the probe IS the half-open mechanism (no separate state). `isBreakerOpen()` getter for consumers.
- `apps/beadboard/src/core/beads-change-watcher.ts` — `readIssues` checks `client.isBreakerOpen()` and routes directly to JSONL fallback when open.
- `apps/gitboard/src/api/routes/internal-dolt-health.ts` (NEW) — `/api/internal/dolt-health` returns `{ pool_size, idle_timeout_ms, probe_interval_ms, breaker_threshold, backoff_min_ms, backoff_max_ms, query_timeout_ms, consecutiveFailures, breakerOpenUntil, latencies: number[] }`. No credentials or query bodies.
- `apps/gitboard/src/types/realtime.ts` — `BeadsSourceHealth` extended with `healthy: boolean`.

### Watcher perf (forge-3mdw → commit e2e8aae) — orchestrator-manual

- `apps/beadboard/src/core/beads-change-watcher.ts` — `lastCommitHash` map per project; `poll()` early-returns with `source_health` emit when hash matches previous AND a snapshot exists (no `readSnapshot` call → no DoltClient queries → ~2ms tick).
- `apps/beadboard/src/core/dolt-client.ts` — `getIssues` + `getClosedIssues` now use `hydrateIssues(rows)`: one issues SELECT + three batched IN-clause queries in parallel (`Promise.all([deps, related, labels])`) replacing the per-row N+1 fan-out. Single-issue `getIssue` keeps the per-row `toIssue` path. Optional tables (`issue_related`, `labels`) wrap their IN-clause queries in try/catch — absent from some Dolt schemas, silently return empty map.
