## Code Changes

### Dashboard stability and identity

- `apps/gitboard/src/dashboard/components/beads/BeadsRepoView.tsx` preserves visible issue data during refresh instead of showing destructive skeleton/empty states.
- `apps/gitboard/src/dashboard/components/beads/IssueFeed.tsx` uses stable virtual row keys and emits render-state telemetry so blank/skeleton/empty transitions can be measured.
- `apps/gitboard/src/dashboard/components/shell/MainPane.tsx`, `apps/gitboard/src/dashboard/hooks/useRepoTree.ts`, `apps/gitboard/src/dashboard/stores/shell.ts`, and `apps/gitboard/src/types/shell.ts` preserve selected repo state, carry Beads project identity, avoid leaking GitHub-only repo selections into Console, and match repo/project aliases more robustly.
- `apps/gitboard/src/dashboard/hooks/useInFlightJobs.ts` preserves prior in-flight jobs when a stale empty response arrives.
- `apps/gitboard/src/dashboard/lib/resource.ts` now supports stale-preserving resource invalidation so background refresh does not erase visible state.

### Graph backend and UI

- `apps/gitboard/src/api/routes/graph.ts` returns actionable degraded `source_health` for missing/unknown project and avoids presenting fresh empty graphs when the source is unavailable.
- `apps/gitboard/src/core/graph-dao.ts` reuses project scans on scoped refresh, caches final graph snapshots for short warm re-navigation, returns stale non-empty graph data while refreshing expired issue cache, and avoids expired empty issue-cache short-circuits.
- `apps/gitboard/src/core/jsonl-reader.ts` reads live `.beads/issues.jsonl` before legacy backup JSONL files.
- `apps/gitboard/src/dashboard/hooks/useGraphData.ts` increases graph TTL, avoids forced refresh during stale-empty retry, and ignores invalidation noise that is not scoped to the active graph project.
- `apps/gitboard/src/dashboard/pages/console/Graph.tsx` surfaces graph source-health messages to make degraded states actionable.

### Beads, specialists, and telemetry APIs

- `apps/beadboard/src/core/beads-change-watcher.ts` logs richer watcher snapshots, issue detection, batch publication, project-scoped batch summaries, and overflow sync hints.
- `apps/beadboard/src/api/routes/beads.ts` logs `beads.issues.response` for issue-list telemetry.
- `apps/gitboard/src/api/routes/specialists.ts` logs job-by-bead and in-flight responses, with freshness/epoch context needed to diagnose stale clears.
- `apps/gitboard/src/api/routes/internal-logs.ts` adds client telemetry ingestion for same-origin dashboard events.
- `apps/gitboard/src/dashboard/lib/client-log.ts` centralizes client-side log submission.

### Tests

- Added/updated targeted coverage for feed identity, repo/project matching, Beads stability, graph cache behavior, graph route source health, graph data refresh semantics, and shell/repo state behavior.
- Targeted tests passed during the session: `tests/api/specialists.cache.test.ts`, `tests/dashboard/components/beads/IssueFeed.identity.test.ts`, `tests/core/beads-change-watcher.test.ts`, `tests/api/routes/graph.test.ts`, and `tests/core/graph-dao.cache.test.ts`.
