## Code Changes

Two coherent commits landed via two PRs. The skeleton's auto-diff is misleading because it ran from the `xt/1ers` worktree (which had not pulled the day's merges yet); the real change inventory is:

**Commit `4a13af7` — observability watcher hot-loop + shared-server routing (PR 12, on top of `1aaa6f5`):**
- `apps/gitboard/src/server/observability/attach-pool.ts` — added a `dead: Map<dbPath, {mtimeMs, reason}>` keyed cache. Probes/attaches that fail get cached with the current mtime; the file-watcher debounce can no longer trigger an infinite re-probe loop on the same dead db. Cache entry drops if the file is rewritten (newer mtime), so legit schema upgrades still get picked up. Only logs first time a given (dbPath, reason) pair is seen dead.
- `apps/gitboard/src/core/project-scanner.ts` — detects `dolt.shared-server: true` in per-project `config.yaml`, reads the actual port from `~/.beads/shared-server/dolt-server.port`, and falls back to `metadata.json`'s `dolt_database` when no per-project port is set. Required for `~/dev/specialists` and other repos that don't run their own sql-server.

**Commit `276abdd` — graph view restructure (PR 13):**
- `apps/gitboard/src/dashboard/pages/console/graph/clusters.ts` (new) — `partitionGraph(data, specialists, options)` does union-find over `STRUCTURAL_EDGE_TYPES`, splits nodes into WIP / clusters / orphans / buckets, sorts clusters by P0 → running → size → edges. `nameCluster()` picks an epic id when shared, else longest-blocks-chain endpoints, else `+N nodes` fallback.
- `apps/gitboard/src/dashboard/pages/console/graph/agent-roles.ts` (new) — role → category mapping (`exec`/`review`/`test`/`research`/`orch`) used for agent-chip styling.
- `apps/gitboard/src/dashboard/pages/console/Graph.tsx` — full rewrite. Layout is `g-app` flex column with `overflow: hidden`: header (with three pill toggles for `parent-child` / `related` / `deferred`), optional NOW strip for running WIP, `g-main` grid (clusters | orphans) consuming `flex: 1`, optional buckets row, pinned `g-foot`. Each cluster pane renders nodes as variable-width HTML divs absolutely positioned inside its canvas; SVG renders edges only.
- `apps/gitboard/src/dashboard/pages/console/graph/layout.ts` — accepts per-node `nodeWidths` map; `anchorY` distributes multi-edge anchors across the node height with 4 px insets, sorted by other endpoint's vertical position; reverse / same-layer edges detour right and land on the target's right edge with the correct width math.
- `apps/gitboard/src/dashboard/pages/console/graph/GraphSvg.tsx` — marker defs use `markerUnits="userSpaceOnUse"` with `refX` inside `markerWidth` so arrowheads overlap the target card edge; exports `computeNodeWidth(node, specialist)`.
- `apps/gitboard/src/dashboard/pages/console/graph/edge-styles.ts` — tighter dash patterns (`parent-child` and `related` at `2 2`; `tracks`/`until` at `4 3`) so dotted edges no longer strand arrowheads.
- `apps/gitboard/src/dashboard/styles/globals.css` — new graph block (~370 lines): tokens, header pills, NOW strip, cluster panes, edge labels, orphan sidebar (scrolls inside its own container, no longer `position: sticky`), inline `<details>` buckets, pinned foot. `.g-app` is fixed-height with `overflow: hidden`; `.g-clusters` and `.g-orphans-list` each own their scroll; `.g-buckets` and `.g-foot` are `flex-shrink: 0` so they pin to the bottom and never get cut off by an inner scroll context.
- `apps/gitboard/tests/dashboard/pages/console/graph/Graph.test.tsx` — updated to match the new rendering shape.
